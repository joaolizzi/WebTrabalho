DOM e Interatividade

Subtítulo de exemplo

Seleção, Alteração, Eventos e Manipulação de Elementos

O DOM (Document Object Model) é uma interface que representa documentos HTML de forma estruturada, permitindo que scripts possam acessar, modificar e interagir com os elementos da página. A manipulação do DOM é fundamental para criar páginas web dinâmicas e interativas, tornando a experiência do usuário muito mais rica.

1. Seleção de Elementos

O que é?
Selecionar elementos é o primeiro passo para manipular qualquer parte da página. Com a seleção, conseguimos acessar e modificar textos, estilos, atributos e até adicionar eventos.

Como funciona?
Existem diferentes métodos para selecionar elementos no DOM. Veja as principais diferenças:
  • getElementById: retorna um único elemento com o id informado.
  • getElementsByClassName: retorna uma coleção (HTMLCollection) com todos os elementos que possuem a classe informada.
  • getElementsByTagName: retorna uma coleção (HTMLCollection) com todos os elementos de uma determinada tag.
  • querySelector: retorna apenas o primeiro elemento que corresponde ao seletor CSS informado.
  • querySelectorAll: retorna todos os elementos que correspondem ao seletor CSS, em uma NodeList.
Destaques:
  • Elemento único vs. Coleção: getElementById e querySelector retornam apenas um elemento. Os outros retornam coleções.
  • Seletores CSS: querySelector e querySelectorAll aceitam qualquer seletor CSS.
  • HTMLCollection vs NodeList: getElementsByClassName e getElementsByTagName retornam HTMLCollection (ao vivo), enquanto querySelectorAll retorna NodeList (estática).

2. Alterar Conteúdo e Estilos

O que é?
Alterar conteúdo e estilos permite atualizar informações exibidas na página e mudar a aparência dos elementos em tempo real, sem recarregar.

Como funciona?
- innerText altera apenas o texto do elemento.
- innerHTML permite inserir HTML, como negrito, itálico, listas, etc.
- style modifica propriedades CSS diretamente pelo JavaScript.

Quando usar?
Sempre que precisar atualizar informações, destacar algo visualmente ou responder a ações do usuário.
  • innerText: Altera apenas o texto do elemento, ignorando qualquer HTML.
  • innerHTML: Permite inserir HTML dentro do elemento, possibilitando formatações.
  • style: Permite alterar estilos CSS diretamente pelo JavaScript.

No exemplo abaixo, mudamos o texto de um parágrafo, inserimos HTML em outro e alteramos a cor de fundo de uma caixa:

// innerText document.getElementById('texto').innerText = "Texto alterado!"; // innerHTML document.getElementById('html').innerHTML = "<strong>HTML alterado com negrito</strong>"; // style document.getElementById('caixa').style.backgroundColor = "lightblue";

3. Eventos (click, keyup, submit)

O que é?
Eventos são ações que acontecem na página, como cliques, digitação, envio de formulários, entre outros. Podemos "ouvir" esses eventos e executar funções quando eles ocorrem.

Como funciona?
Usamos addEventListener para associar uma função a um evento específico de um elemento.

Quando usar?
Sempre que quiser tornar a página interativa, respondendo a ações do usuário.
  • click: Disparado quando o usuário clica em um elemento.
  • keyup: Disparado quando o usuário solta uma tecla após digitar em um campo.
  • submit: Disparado ao enviar um formulário.

Veja exemplos práticos de cada evento:

Evento Click

Explicação:
O evento click é um dos mais usados. Ele permite executar uma ação sempre que o usuário clicar em um botão, link ou qualquer outro elemento clicável. No exemplo, ao clicar no botão, um alerta aparece na tela.
document.getElementById('btnClick').addEventListener('click', function() { alert("Você clicou no botão!"); });

Evento Keyup

Explicação:
O evento keyup é disparado quando o usuário solta uma tecla após digitar em um campo. Isso permite, por exemplo, mostrar sugestões, validar campos em tempo real ou atualizar informações instantaneamente. No exemplo, o texto digitado aparece logo abaixo do campo.
document.getElementById('inputKeyup').addEventListener('keyup', function(e) { document.getElementById('resultadoKeyup').innerText = "Você digitou: " + e.target.value; });

Evento Submit

Explicação:
O evento submit é disparado ao enviar um formulário. Usamos preventDefault() para evitar que a página recarregue, permitindo validar dados ou enviar informações via JavaScript. No exemplo, um alerta confirma o envio.
document.getElementById('formulario').addEventListener('submit', function(e) { e.preventDefault(); alert("Formulário enviado com sucesso!"); });
Resumo:
O uso de eventos é fundamental para criar páginas responsivas, que reagem às ações do usuário de forma instantânea. Eles são a base de toda interatividade em aplicações web modernas.

4. Criar e Remover Elementos

O que é?
Criar e remover elementos dinamicamente permite que a página cresça ou diminua conforme a interação do usuário, sem precisar recarregar.

Como funciona?
- createElement cria um novo elemento HTML.
- appendChild adiciona esse elemento como filho de outro.
- removeChild remove um elemento filho.

Quando usar?
Sempre que precisar adicionar itens em listas, criar cards, mensagens, ou remover elementos da tela.
  • createElement: Cria um novo elemento HTML via JavaScript.
  • appendChild: Adiciona um elemento como filho de outro elemento.
  • removeChild: Remove um elemento filho.

No exemplo abaixo, ao clicar no botão, um novo item é adicionado à lista:

document.getElementById('btnCriar').addEventListener('click', function() { let novoItem = document.createElement('li'); novoItem.innerText = "Novo item"; document.getElementById('lista').appendChild(novoItem); });
    Resumo:
    Com essas técnicas, é possível criar aplicações web dinâmicas, como listas de tarefas, chats, sistemas de comentários e muito mais, tudo sem recarregar a página. Isso deixa a experiência do usuário mais fluida e moderna.

    Resumo

    Manipular o DOM é uma das habilidades mais importantes para quem desenvolve para web. Com a seleção de elementos, alteração de conteúdo e estilos, uso de eventos e criação dinâmica de elementos, é possível transformar páginas estáticas em experiências ricas e interativas para o usuário.

    Pratique bastante esses conceitos e experimente criar suas próprias interações para fixar o aprendizado!
